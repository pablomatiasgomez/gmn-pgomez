2. guessnumber

Tries to guess a player’s actual number, and returns how many digits are correct, including it’s position, how many are correct, but in other position, and how many are incorrect. This command also return a time to wait before next attempt. If a second call is made before that waiting time, the player will be penalized.

Another thing to keep in mind, is that the numberId hashes, are the same for any user. For example, if Player1 sets 1234 as his own number, and Player2 chooses the same number, both numberId hashes will have the same value.

Request

Method

URL            

GET

/play/guessnumber/<privateUuid>/<publicUuid>/<number>

privateUuid

 privateUuid  UUID of the current player obtained from the player/register command.

publicUuid

 publicUuid  UUID of the player obtained from the player/board command.

number

4 digits number. They can be repeated

Glossary

Conventions

  1. Client - Client application.
  2. Status - HTTP status code of response.
  3. All the possible responses are listed under ‘Responses’ for each method. Only one of them is issued per request server.
  4. All response are in JSON format.
  5. All request parameters are mandatory unless explicitly marked as [optional]
  6. The type of values accepted for a request parameter are shown the the values column like this [10|<any number>] .The | symbol means OR. If the parameter is [optional], the default value is shown in blue bold text, as 10 is written in [10|<any number>].

Status Codes

All status codes are standard HTTP status codes. The below ones are used in this API.

2XX - Success of some kind

4XX - Error occurred in client’s part

5XX - Error occurred in server’s part
